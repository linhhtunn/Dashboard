# Pipeline Publisher

Replay publisher for files generated by `simulator.core`.

## Dry Run

```bash
cd backend
python -m simulator.pipeline.publisher.replay_generated_data \
  --patient-id P005 \
  --dry-run \
  --limit 5
```

## Publish

Set `RABBITMQ_URL` in the environment, then run:

```bash
cd backend
python -m simulator.pipeline.publisher.replay_generated_data --patient-id P005
```

Use `--no-declare` when the RabbitMQ topology already exists and should not be
declared by the publisher.

The publisher supports simulator v2 patient-directory output:

```text
simulator/output/P005/continuous.jsonl
simulator/output/P005/steps_event.jsonl
simulator/output/P005/stress.jsonl
simulator/output/P005/ppi_batch.jsonl
simulator/output/P005/motion_batch.jsonl
simulator/output/P005/bp_triggered.jsonl
simulator/output/P005/spo2_triggered.jsonl
simulator/output/P005/battery.jsonl
simulator/output/P005/ecg_triggered.jsonl
simulator/output/P005/sleep_timeline.json
simulator/output/P005/daily_metrics.json
```

It also keeps compatibility with the older flat files such as
`wearable_continuous_P005_24h.jsonl` when `--suffix P005_24h` is used.

Publish selected streams:

```bash
python -m simulator.pipeline.publisher.replay_generated_data \
  --patient-id P005 \
  --streams wearable_continuous wearable_ppi_batch wearable_motion_batch wearable_spo2_triggered \
  --limit 120 \
  --delay-seconds 0 \
  --no-declare
```

The publisher depends on shared broker helpers from `backend/rabbit_mq`.
